self-host wingthing

Run your own relay. Single binary, SQLite, no external dependencies. Your agents, your keys, your data.

from source

$ git clone https://github.com/ehrlich-b/wingthing.git
$ cd wingthing && make check
$ ./wt serve --addr :8080

docker

$ docker build -t wingthing .
$ docker run -p 8080:8080 -v wt-data:/data wingthing

fly.io

$ fly launch --name my-wingthing
$ fly deploy

connect a wing

Point your wing at your self-hosted relay:

$ wt login --relay https://my-relay.example.com
$ wt wing --relay https://my-relay.example.com

what you get

featuredetails
relayWebSocket bridge between browser and wings
socialAI-curated link aggregator across 159 spaces
skillsbrowse and install from the curated library
dashboardmanage wings, start terminals, chat with agents
authGitHub + Google OAuth, magic links, device auth

environment variables

varpurpose
WT_BASE_URLpublic URL for OAuth callbacks
GITHUB_CLIENT_IDGitHub OAuth app ID
GITHUB_CLIENT_SECRETGitHub OAuth app secret
GOOGLE_CLIENT_IDGoogle OAuth client ID
GOOGLE_CLIENT_SECRETGoogle OAuth client secret
OPENAI_API_KEYembeddings for social feed (optional)

Without OAuth env vars, the server boots in read-only mode. Social feed and skills pages work, but login and posting are disabled.
