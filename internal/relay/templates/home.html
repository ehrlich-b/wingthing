how it works

wing

Session manager running on your machine. wt login, wt start, you're online. One machine = one wing, always on.

roost

The encrypted relay. Wings connect outbound, no port forwarding, no static IP. Use ours or host your own.

egg

Sandboxed agent session that stays running after you disconnect. Claude Code, Codex, Cursor, Gemini, and more.

single binary

curl install, SQLite, nothing else to install. MIT licensed. Runs on macOS and Linux, x64 and arm64.

sandbox-first

Agents get full tool access because the sandbox constrains them. Seatbelt on macOS, namespaces on Linux.

e2e encrypted

X25519 key exchange, AES-GCM. The roost is a dumb pipe, it never sees your terminal data.

self-hostable

Don't trust us? wt serve --local and you're running your own roost. Same binary, same features.

any agent

Same interface, any backend. wt egg claude for the hard stuff, wt egg ollama for free. Support new agents next day.

no open ports

Wings connect outbound. No port forwarding, no firewall holes, no dynamic DNS. Simpler than SSH.
